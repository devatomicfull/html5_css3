Prefixos proprietarios

     .button {
            /* Prefixo para o Firefox. */
            -moz-transition: all 0.2s linear;
            /* Prefixo para o Chorme, Safari, Safari Mobile e Android. */
            -webkit-transition: all 0.2s linear;
            /* Prefixo para o Opera */
            -o-transition: all 0.2s linear;
            /* Versão final, ainda não suportada em todos os browsers*/
            transition: all 0.2s linear;
     }

Muitas propriedades e funcionalidades novas do CSS ainda não estão totalmente definidas ou implementadas nos navegadores. Mesmo assim, podemos usá-las em versões “experimentais”. Para isso, cada navegador libera essas propriedades usando um prefixo próprio, indicando que ainda não são a versão final. Na teoria isso é útil, mas na prática virou um problema: o desenvolvedor precisa escrever a mesma regra de CSS várias vezes, cada uma com um prefixo diferente, apenas para garantir compatibilidade entre os navegadores.

Essas propriedades enquanto elas são defidas e refinadas. O objetivo é, eventualmente, não precisar mais das versões prefixadas e se utilizar apenas a versão canônica das propriedades

Diferença simples entre pseudo-classe e pseudo-elemento

Tipo O que faz Sintaxe
Pseudo-classe Muda o estado do elemento : (ex: :hover, :active)
Pseudo-elemento Cria ou estiliza uma parte do elemento :: (ex: ::before, ::after)

Dica para nunca esquecer

Pseudo-CLASSE usa 1 dois-pontos: :hover → classe = estado
Pseudo-ELEMENTO usa 2 dois-pontos: ::before → elemento = parte criada
